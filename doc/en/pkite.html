🚀 1. Quick Start

# Clone repository (🌟Stars appreciated)
git clone https://github.com/chenwumm/PaperKite/

# Create project directory
mkdir my_project && cd my_project

# Initialize repository 🌱
python ~/PaperKite/pkite.py init

# Add all files ➕
python ~/PaperKite/pkite.py add .

# First commit 💌
python ~/PaperKite/pkite.py commit "Hello PaperKite!"

📚 2. Core Features

📦 Repository Initialization

python ~/PaperKite/pkite.py init
  • Creates hidden .pkite directory
  • Generates HEAD and index files
  • ❗️ Warns if repository already exists

➕ File Tracking

# Add single file
python ~/PaperKite/pkite.py add README.md
# Add entire directory
python ~/PaperKite/pkite.py add .
  • Stores file contents using SHA-1 hash
  • Automatically ignores .pkite directory

❓ 3. About the Project

Why the name "PaperKite"?

PaperKite translates to "paper kite," where "paper" represents the project's lightweight nature, and "kite" symbolizes its flexibility and ease of use.

What are the advantages?

1. Approximately 160 lines of code, compact size.
2. Retains core features of other version control systems (like Git, SVN) such as initialization and commits.

What are the disadvantages?

1. Compared to Git and other VCS, PKite lacks branch concepts, code diff functionality, etc.
2. Cannot view commit history.
3. Not suitable for large projects or team collaboration.

What scenarios is it suitable for?

1. Suitable for personal development and small to medium projects.
2. Suitable for learning the basic principles of VCS (Version Control Systems).
